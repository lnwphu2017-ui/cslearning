6. Digital and Boolean Algebra
บทที่ 1 ระบบตัวเลขและพีชคณิตบูลีน (Number Systems and Boolean Algebra) 
**พีชคณิตบูลีน (Boolean Algebra)** //คณิตศาสตร์ของความจริงในขณะที่พีชคณิตทั่วไปจัดการกับตัวเลขที่มีค่าไม่สิ้นสุด พีชคณิตบูลีนจัดการกับค่าเพียงสองค่าคือ True (1) และ False (0) นี่คือหัวใจที่ใช้ในการออกแบบวงจรประมวลผล (CPU) และการเขียนเงื่อนไขในโปรแกรม
1 AND (cdot) จะเป็นจริงก็ต่อเมื่ออินพุตทุกตัวเป็นจริง เปรียบเสมือนการคูณ (1 \cdot 1 = 1, แต่ 1 \cdot 0 = 0)
2 OR (+) จะเป็นจริงขอเพียงแค่อินพุตตัวใดตัวหนึ่งเป็นจริง เปรียบเสมือนการบวก (1 + 0 = 1)
3 NOT (overline{A}) คือการกลับค่า (Inversion) จาก 1 เป็น 0 หรือ 0 เป็น 1 //

**ลอจิกเกต (Logic Gates)ชิ้นส่วนเลโก้ของโลกดิจิทัล** //เมื่อเรานำพีชคณิตบูลีนมาสร้างเป็นอุปกรณ์กายภาพ เราจะได้ "ลอจิกเกต" ซึ่งเป็นส่วนประกอบพื้นฐานของไอซี (IC) และไมโครโพรเซสเซอร์
1 เกตพื้นฐาน AND, OR, NOT
2 เกตประยุกต์ NAND / NOR เป็น "Universal Gates" ที่สามารถสร้างเกตชนิดอื่นได้ทุกชนิด
3 XOR (Exclusive OR) จะเป็นจริงก็ต่อเมื่ออินพุตทั้งสองค่า "ต่างกัน" เท่านั้น ซึ่งสำคัญมากในวงจรบวกเลข //

**ตารางค่าความจริง (Truth Tables) แผนผังการตัดสินใจ** //เพื่อความชัดเจนในการวิเคราะห์ตรรกะที่ซับซ้อน เราจะใช้ "ตารางค่าความจริง" เพื่อแสดงผลลัพธ์ที่เป็นไปได้ทั้งหมดจากอินพุตทุกรูปแบบ
1 การใช้งาน ช่วยให้นักพัฒนาสามารถลดรูปสมการตรรกะที่ยุ่งเหยิงให้สั้นลงได้ (Optimization)
2 ผลลัพธ์ ส่งผลโดยตรงต่อความเร็วในการประมวลผลของฮาร์ดแวร์และการใช้ทรัพยากรที่น้อยลง //

**ทำไมบทนี้จึงเป็นจุดเริ่มต้นที่สำคัญ?** //การเข้าใจตรรกะในระดับบิต (Bitwise Logic) มีผลมหาศาลในการทำงานจริงมากกว่าแค่การคำนวณ
1 Low-level Optimization การเขียนโค้ดที่ทำงานกับบิตโดยตรงช่วยลดภาระของ CPU ได้มหาศาล เช่น ในงานกราฟิกหรือการเข้ารหัส
2 AI & Decision Making พื้นฐานของ AI Agents คือการตัดสินใจตามเงื่อนไข (Conditionals) ซึ่งล้วนขยายผลมาจากตรรกะพื้นฐานเหล่านี้
3 System Reliability การออกแบบระบบที่ไร้บั๊กเริ่มจากการวางตรรกะที่รัดกุมตั้งแต่ระดับพื้นฐาน //

บทที่ 2 การลดรูปวงจรลอจิก (Logic Simplification)
**ทำไมเราต้องลดรูปวงจร?** //การใช้เกต (Gate) น้อยลงส่งผลดีใน 4 มิติหลักที่ส่งผลต่อคุณภาพของฮาร์ดแวร์โดยตรง
1 Cost ลดจำนวนทรานซิสเตอร์ ทำให้ชิปราคาถูกลง
2 Size บีบอัดวงจรให้เล็กลง เพื่อใส่ฟีเจอร์อื่นเพิ่มได้มากขึ้นในพื้นที่เท่าเดิม
3 Speed กระแสไฟฟ้าเดินทางสั้นลง ลดความหน่วง (Propagation Delay) ทำให้ประมวลผลเร็วขึ้น
4 Power กินไฟน้อยลงและเกิดความร้อนน้อยลง ซึ่งสำคัญมากสำหรับอุปกรณ์พกพา //

**เครื่องมือที่ 1 ทฤษฎีบทของบูลีน (Boolean Theorems)**//การลดรูปด้วยสมการคณิตศาสตร์คือการใช้กฎต่างๆ เพื่อยุบรวมพจน์ที่ซ้ำซ้อน
1 กฎพื้นฐาน เช่น กฎการสลับที่ (Commutative) หรือกฎการเปลี่ยนกลุ่ม (Associative)
2 De Morgan's Theorem กฎที่ทรงพลังในการกระจายเครื่องหมาย NOT เข้าไปในวงเล็บ
3 ตัวอย่างการลดรูป การจำรูปแบบเฉพาะเช่นA + \overline{A}B = A + Bช่วยให้นักออกแบบมองเห็นทางลัดในสมการตรรกะได้ทันที //

**เครื่องมือที่ 2 แผนผังคาร์โนห์ (Karnaugh Map / K-Map)**//หากสมการบูลีนคือการคำนวณ K-Map ก็คือ "กราฟิกดีไซน์" ของตรรกะ โดยจัดเรียงค่าความจริงในรูปแบบ Gray Code ที่แต่ละช่องต่างกันเพียง 1 บิต
1 ศิลปะของการจับกลุ่ม (Grouping) เราจะนำเลข 1 หรือเลข 0 มาวางลงในตาราง แล้วทำการ "วงกลม" จับกลุ่มสมาชิกในจำนวนเลขยกกำลังสอง (1, 2, 4, 8, 16)
2 การตัดตัวแปร ยิ่งวงกลมมีขนาดใหญ่ ตัวแปรจะยิ่งถูกตัดทิ้งไปได้มากขึ้น
3 การม้วนตาราง ตารางนี้มีความพิเศษคือมัน "ม้วน" เข้าหากันได้ ทำให้เราสามารถจับกลุ่มจากขอบตารางที่อยู่คนละฝั่งกันได้ //

**SOP vs. POS** เลือกอาวุธให้ถูกทาง//ในการเลือกรูปแบบผลลัพธ์สุดท้าย เราต้องพิจารณาจากจำนวนเลข 0 และ 1 ในตารางค่าความจริง1 Sum of Products (SOP) เน้นที่ผลลัพธ์เป็น 
1 โดยนำกลุ่มที่คูณกัน (AND) มาบวกกัน (OR) เป็นรูปแบบที่นิยมที่สุด
2 Product of Sums (POS) เน้นที่ผลลัพธ์เป็น 0 โดยนำกลุ่มที่บวกกัน (OR) มาคูณกัน (AND) มักใช้เมื่อตารางความจริงมีเลข 0 น้อยกว่าเลข 1 มากๆ //

บทที่ 3 วงจรตรรกะแบบผสม (Combinational Logic Circuits)
**วงจรบวกเลข (Adders) หัวใจของการคำนวณ**//คอมพิวเตอร์ไม่ได้บวกเลขเก่งมาตั้งแต่เกิด แต่มันใช้ตรรกะ XOR และ AND มาทำงานร่วมกันเพื่อให้ได้ผลลัพธ์
1 Half Adder (ตัวบวกครึ่ง) รับอินพุต 2 บิต (A, B) แล้วคายผลลัพธ์เป็น ผลรวม (Sum) และ ตัวทด (Carry) แต่มีข้อจำกัดคือ "รับตัวทดจากหลักก่อนหน้าไม่ได้"
2 Full Adder (ตัวบวกเต็ม) คือการอัปเกรดที่สมบูรณ์แบบ มันรับอินพุตได้ 3 ตัว (A, B และ C_{in}) ทำให้เราสามารถนำวงจรนี้มาวางเรียงต่อกันเพื่อบวกเลขกี่บิตก็ได้ในจักรวาล //

**Multiplexer (Mux) สวิตช์สับรางข้อมูล** //ลองจินตนาการถึงทางรถไฟที่มีรางหลายสายวิ่งเข้ามา แต่มีรางเดียวที่มุ่งหน้าเข้าสู่สถานี
1 หน้าที่ ทำหน้าที่เป็นตัวเลือก (Selector) ว่าจะให้ข้อมูลจากเส้นทางไหนไหลผ่านไปสู่ผลลัพธ์
2 การตัดสินใจ ใช้สัญญาณควบคุม (Control Signal) เป็นตัวตัดสินใจว่าจะเลือกสายไหน
3 การใช้งาน ใช้ในหน่วยประมวลผลเพื่อเลือกว่าจะเอาข้อมูลจาก Register ตัวไหนมาคำนวณในรอบนี้ //

**Decoder & Encoder นักถอดรหัสและนักแปรรหัส** //สองวงจรนี้ทำงานในทิศทางตรงกันข้ามเพื่อจัดการกับความหมายของข้อมูล
1 Decoder (ตัวถอดรหัส) เปลี่ยนรหัสฐานสองให้กลายเป็น "ตำแหน่ง" หรือ "คำสั่ง" ที่ชัดเจน เช่น รับ 00 เข้ามาแล้วเปิดไฟดวงที่ 1 สำคัญมากในการถอดรหัสคำสั่งใน CPU
2 Encoder (ตัวเข้ารหัส) รับสัญญาณจากหลายแหล่ง แล้วสรุปกลับมาเป็นรหัสฐานสองที่กระชับ เพื่อประหยัดพื้นที่ในการส่งข้อมูล //

**ก้าวสู่ ALU (Arithmetic Logic Unit)** //เมื่อเรานำ Adder, Mux, และ Decoder มารวมร่างกัน เราจะได้ ALU ซึ่งเป็นเหมือน "โรงงานคำนวณ" ภายใน CPU
1 ความสามารถ สามารถบวก ลบ และเปรียบเทียบตรรกะได้ทุกอย่างตามที่เราสั่งการ
2 จุดสังเกต ในวงจรแบบผสมนี้ หากคุณถอดอินพุตออก ผลลัพธ์จะหายไปทันที (เพราะไม่มี Memory) นี่จึงเป็นเหตุผลว่าบทถัดไปเราจะคุยกันเรื่อง "การจดจำ" ครับ //

บทที่ 4 วงจรตรรกะแบบลำดับ (Sequential Logic Circuits)
**พลังของ Feedback Loop ความลับของการจดจำ** //หัวใจที่ทำให้วงจรตรรกะเริ่มจำค่าได้คือการทำ Feedback หรือการนำ "ผลลัพธ์" วนกลับมาเป็น "อินพุต" อีกครั้งหนึ่ง
1 สถานะ Bistable เมื่อสัญญาณวนกลับมาหาตัวเอง มันจะค้างอยู่ที่ค่าใดค่าหนึ่ง (0 หรือ 1) ตราบเท่าที่ยังมีไฟเลี้ยง
2 จุดกำเนิดหน่วยความจำ นี่คือพื้นฐานของ Latch และ Flip-Flop ซึ่งทำหน้าที่เก็บข้อมูลขนาด 1 บิต //

**ตระกูล Flip-Flops: ขุนพล 1 บิต** //Flip-Flop คือหน่วยพื้นฐานที่สุดของหน่วยความจำใน CPU และ RAM โดยแต่ละประเภทมีวิธีการ "จำ" ที่ต่างกัน
1 SR Flip-Flop (Set-Reset) มีปุ่ม "ตั้งค่า" (Set) และ "ล้างค่า" (Reset) แต่ห้ามป้อนค่า 1 พร้อมกัน (Forbidden State)
2 D Flip-Flop (Data) พระเอกของวงการ! จะจำค่าที่ป้อนเข้าที่ขา D ทันทีเมื่อถึงจังหวะสัญญาณนาฬิกา นิยมใช้ทำ Registers มากที่สุด
3 JK Flip-Flop รุ่นอัปเกรดของ SR ที่แก้ปัญหาการกดพร้อมกัน โดยการทำให้มัน "สลับค่า" (Toggle) แทน
4 T Flip-Flop (Toggle) ใช้สำหรับสลับค่าไปมา (0 เป็น 1 หรือ 1 เป็น 0) มักใช้ในวงจรนับ (Counters) //

**สัญญาณนาฬิกา (Clock) จังหวะหัวใจของเครื่องจักร** //ถ้า Flip-Flop คือเซลล์สมอง สัญญาณนาฬิกา (Clock Signal) ก็คือจังหวะการเต้นของหัวใจที่คอยควบคุมจังหวะการทำงานครับ
1 Synchronous วงจรจะเปลี่ยนสถานะพร้อมกันตามจังหวะ "ขอบขาขึ้น" (Rising Edge) หรือ "ขอบขาลง" (Falling Edge)
2 ความเร็ว CPU (GHz) เมื่อพูดว่า CPU เร็ว 4.0 GHz หมายความว่าสัญญาณนาฬิกาเต้นถึง 4,000 ล้านครั้งต่อวินาที และ Flip-Flop นับล้านตัวจะอัปเดตค่าตามจังหวะนี้ //

**จาก Flip-Flop สู่หน่วยความจำมหาศาล** //เมื่อนำ Flip-Flop จำนวนมากมาวางเรียงต่อกันและทำงานร่วมกัน เราจะได้อุปกรณ์ที่ซับซ้อนขึ้น
1 Registers หน่วยความจำที่เร็วที่สุด อยู่ใน CPU เพื่อเก็บข้อมูลที่กำลังประมวลผล (ใช้ D Flip-Flop)
2 Counters วงจรนับเลขที่ใช้ในระบบดิจิทัล (ใช้ JK หรือ T Flip-Flop)
3 SRAM (Static RAM) หน่วยความจำแคช (Cache) ที่รวดเร็วและไม่ต้อง Refresh ข้อมูลบ่อยๆ //

บทที่ 5 การออกแบบรีจิสเตอร์และตัวนับ (Design of Registers and Counters)
**รีจิสเตอร์ (Registers)** //ขุนพลความเร็วสูงประจำ CPUหากเรานำ D Flip-Flop มาวางเรียงกัน $n$ ตัว เราจะได้รีจิสเตอร์ขนาด n บิต ในลำดับชั้นของหน่วยความจำ (Memory Hierarchy) รีจิสเตอร์คือ "จุดที่เร็วที่สุด" เพราะอยู่ติดกับหน่วยคำนวณ (ALU) มากที่สุด
1 Parallel Load ความสามารถในการรับข้อมูลทุกบิตเข้ามาเก็บพร้อมกันทันที มักใช้ใน CPU เพื่อพักค่าตัวแปรชั่วคราวระหว่างประมวลผล
2 ความสำคัญ ช่วยให้ ALU ไม่ต้องรอข้อมูลนานๆ จาก RAM ที่อยู่ไกลและช้ากว่า //

**Shift Registers สายพานลำเลียงบิต** //รีจิสเตอร์ประเภทนี้ไม่ได้แค่ "จำ" แต่ยังสามารถ "เลื่อน" ข้อมูลไปทางซ้ายหรือขวาได้ด้วย โดยการต่อเอาต์พุตของ Flip-Flop ตัวแรกเข้ากับอินพุตของตัวถัดไปเป็นทอดๆ
1 Serial to Parallel Conversion รับข้อมูลมาทีละบิต (เช่น จากสาย USB) แล้วสะสมจนครบก่อนจะส่งเข้าประมวลผลแบบขนาน
2 คณิตศาสตร์ความเร็วสูง การเลื่อนบิตไปทางซ้าย 1 ครั้ง เท่ากับการ คูณสอง และการเลื่อนไปทางขวา 1 ครั้ง เท่ากับการ หารสอง ซึ่งเร็วกว่าการใช้หน่วยคำนวณปกติมากครับ //

**วงจรนับ (Counters) ผู้คุมจังหวะของระบบ** //วงจรนับคือวงจรแบบลำดับที่เดินหน้าผ่าน "สถานะ" ต่างๆ ตามจังหวะสัญญาณนาฬิกา ซึ่งแบ่งออกเป็น 2 ประเภทใหญ่ตามลักษณะการจ่ายนาฬิกา//

**A. Asynchronous (Ripple) Counters**//1 การทำงาน สัญญาณนาฬิกาจะส่งไปที่ตัวแรกตัวเดียว ส่วนตัวถัดไปใช้เอาต์พุตจากตัวก่อนหน้าเป็นตัวกระตุ้น (เหมือนโดมิโน)2 ข้อดี/ข้อเสีย ออกแบบง่ายแต่มีความหน่วงสะสม (Propagation Delay) หากนับเลขหลายบิตที่ความเร็วสูงอาจเกิดความผิดพลาดได้ //

**B. Synchronous Counters** //1 การทำงาน ทุก Flip-Flop จะได้รับสัญญาณนาฬิกาเส้นเดียวกันและทำงาน พร้อมกัน เป๊ะๆ2 ข้อดี ทำงานได้ที่ความเร็วสูงมาก (High Frequency) เพราะไม่มีการรอกันระหว่างบิต3 ข้อเสีย วงจรซับซ้อนกว่าเพราะต้องใช้ลอจิกเพิ่มเพื่อคุมว่าบิตไหนควรเปลี่ยนค่าในรอบถัดไป //

บทที่ 6 เครื่องจักรสถานะจำกัด (Finite State Machines - FSM)
**องค์ประกอบของ FSM** //แผนภาพสถานะ (State Diagram)การออกแบบระบบที่ซับซ้อน เราจะไม่เริ่มที่การต่อสายไฟ แต่เราจะเริ่มจากการวาด "แผนภาพสถานะ" เพื่อกำหนดพฤติกรรมของระบบ
1 สถานะ (States) แทนด้วยวงกลม คือสภาวะที่ระบบเป็นอยู่ในขณะนั้น (เช่น สถานะ "ปิดไฟ" หรือ "เปิดไฟ")
2 การเปลี่ยนสถานะ (Transitions) แทนด้วยลูกศร คือการย้ายจากสถานะหนึ่งไปอีกสถานะหนึ่งเมื่อมีเหตุการณ์ (Input) มากระตุ้น
3 เงื่อนไข (Conditions) ตัวเลขบนลูกศรที่บอกว่าถ้า Input เป็น 0 หรือ 1 จะให้วิ่งไปทางไหน //

**สองโมเดลหลัก: Moore vs. Mealy** //ในการออกแบบวงจรควบคุม เรามีวิธีคิด 2 รูปแบบที่ให้ผลลัพธ์ต่างกันในเรื่องของจังหวะเวลา (Timing) และจำนวนเกตที่ใช้คุณสมบัติMoore MachineMealy Machineที่มาของ Outputขึ้นอยู่กับ สถานะปัจจุบัน เท่านั้นขึ้นอยู่กับ สถานะปัจจุบัน และ อินพุตสมการตรรกะ$Output = f(Present State)$$Output = f(Present State, Input)$การตอบสนองมักจะช้ากว่า 1 จังหวะนาฬิกาตอบสนองได้ทันทีเมื่ออินพุตเปลี่ยนความซับซ้อนจำนวนสถานะมากกว่า แต่ออกแบบง่ายสถานะน้อยกว่า แต่ลอจิกซับซ้อนกว่า//

**ขั้นตอนการเปลี่ยนความคิดให้เป็นวงจร** //เมื่อเราได้แผนภาพสถานะแล้ว วิศวกรจะทำตามขั้นตอนมาตรฐานเพื่อเปลี่ยนมันให้เป็นฮาร์ดแวร์จริง
1 State Reduction ลดจำนวนสถานะที่ซ้ำซ้อนเพื่อให้วงจรเล็กที่สุด (ใช้ความรู้เรื่อง K-Map มาช่วย)
2 State Assignment แทนชื่อสถานะด้วยรหัสไบนารี (เช่น สถานะ A = 00, B = 01)
3 Transition Table สร้างตารางเพื่อดูว่าต้องใช้ Flip-Flop ตัวไหนและต่อเกตอย่างไร
4 Implementation นำลอจิกเกตมาต่อเข้ากับ Flip-Flop เพื่อสร้าง Control Unit //

**การประยุกต์ใช้ จากตู้กดน้ำสู่หน่วยประมวลผล** //FSM คือพื้นฐานของระบบอัตโนมัติทุกอย่างที่คุณพบเจอในชีวิตประจำวันและโลกของคอมพิวเตอร์
1 ชีวิตประจำวัน ระบบควบคุมลิฟต์, ตู้กดน้ำ (ต้องหยอดเหรียญให้ครบสถานะถึงจะกดน้ำได้), ไฟจราจร
2 สถาปัตยกรรมคอมพิวเตอร์ วงจรควบคุมลำดับคำสั่ง (Instruction Decoder/Sequencer) ภายใน CPU ที่คอยสั่งว่าจังหวะนี้ต้องอ่านข้อมูลจาก RAM และจังหวะถัดไปต้องส่งให้ ALU //

บทที่ 7 หน่วยความจำ และวงจรแปลงสัญญาณ (Memory and ADC/DAC)

**การเชื่อมต่อสองโลก ADC และ DAC** //ในโลกความเป็นจริง ข้อมูลทุกอย่างเป็น Analog (ต่อเนื่อง) แต่คอมพิวเตอร์เป็นระบบ Digital (0 และ 1) เราจึงต้องมีตัวแปลภาษาดังนี้ครับ
ADC (Analog-to-Digital Converter)
ทำหน้าที่เปลี่ยนแรงดันไฟฟ้าที่ต่อเนื่องให้กลายเป็น "รหัสตัวเลข" ผ่าน 2 ขั้นตอนหลัก:
1 Sampling (การสุ่มตัวอย่าง) คือการวัดค่าสัญญาณเป็นระยะๆ ยิ่งสุ่มบ่อย (Sampling Rate สูง) ข้อมูลจะยิ่งเหมือนต้นฉบับ
2 Quantization (การกำหนดระดับ) คือการนำค่าที่วัดได้มาเทียบกับระดับตัวเลขดิจิทัล (Bit Depth) เพื่อแทนค่าแรงดันนั้นๆ
DAC (Digital-to-Analog Converter)
1 หน้าที่ ทำหน้าที่ตรงกันข้าม คือรับ "รหัสตัวเลข" มาเปลี่ยนกลับเป็นแรงดันไฟฟ้าที่ต่อเนื่อง
2 การใช้งาน เพื่อขับเคลื่อนอุปกรณ์ปลายทาง เช่น ลำโพงให้สั่นสะเทือนเป็นคลื่นเสียง หรือปรับความสว่างหน้าจอ //

**สถาปัตยกรรมหน่วยความจำ (Memory Architecture)** //เมื่อแปลงข้อมูลเป็นดิจิทัลแล้ว เราต้องการ "คลังสินค้า" เพื่อจัดเก็บ ซึ่งแบ่งตามลักษณะการรักษาข้อมูลได้ 2 ประเภทหลักครับ
RAM (Random Access Memory)
1 นิยาม หน่วยความจำประเภท Volatile (ระเหยได้) ข้อมูลจะหายไปทันทีเมื่อไม่มีไฟเลี้ยง
2 การใช้งาน ใช้เก็บข้อมูลที่ CPU กำลังประมวลผล (Working Memory) เพราะมีความเร็วสูงมาก
ROM (Read-Only Memory)
1 นิยาม หน่วยความจำประเภท Non-Volatile (คงสภาพ) ข้อมูลยังอยู่แม้ปิดเครื่อง
2 วิวัฒนาการ จาก ROM ที่เขียนได้ครั้งเดียว พัฒนามาเป็น Flash Memory (เช่น SSD) ที่ใช้กันแพร่หลายในปัจจุบัน //

**ลำดับชั้นของหน่วยความจำ (Memory Hierarchy)** //ในการออกแบบคอมพิวเตอร์ เราต้องแลกเปลี่ยน (Trade-off) ระหว่างความเร็ว ความจุ และราคา จึงเกิดการจัดลำดับชั้นขึ้นครับ
1 Registers เร็วที่สุด อยู่ใน CPU (ใช้เก็บข้อมูลระดับบิตที่กำลังคำนวณ)
2 Cache (SRAM) เร็วมาก ใช้พักข้อมูลที่ CPU เรียกใช้งานบ่อยๆ
3 Main Memory (DRAM/RAM) ขนาดใหญ่ขึ้นมา เพื่อใช้รันโปรแกรมต่างๆ
4 Secondary Storage (SSD/HDD) ช้าที่สุด แต่จุได้มหาศาลและเก็บข้อมูลถาวร//
