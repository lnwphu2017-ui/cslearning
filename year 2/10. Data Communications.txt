10. Data Communications
บทที่ 1: องค์ประกอบสื่อสารข้อมูล (The Foundation of Connectivity)
**องค์ประกอบหลัก 5 ประการของการสื่อสารข้อมูล**
//หัวใจสำคัญของการส่งผ่านข้อมูลจากจุดหนึ่งไปยังอีกจุดหนึ่งประกอบด้วยส่วนประกอบที่ขาดไม่ได้ดังนี้
1 Sender อุปกรณ์ต้นทางที่ทำหน้าที่ส่งข้อมูล เช่น คอมพิวเตอร์ที่คุณใช้พัฒนาโปรเจกต์
2 Receiver อุปกรณ์ปลายทางที่รอรับข้อมูล เช่น Cloud Server ที่จัดเก็บฐานข้อมูล
3 Message ตัวข้อมูลสารสนเทศที่ต้องการส่งผ่าน ไม่ว่าจะเป็นข้อความ รูปภาพ หรือคำสั่งโปรแกรม
4 Medium สื่อกลางหรือทางเดินของข้อมูล มีทั้งแบบใช้สายอย่าง Fiber Optic ที่ให้ความเร็วสูง และแบบไร้สายอย่าง 5G ที่เน้นความคล่องตัว
5 Protocol กฎระเบียบหรือมาตรฐานที่ตกลงกันไว้เพื่อให้ทั้งสองฝั่งคุยกันรู้เรื่อง เช่น HTTP หรือ TCP/IP หากขาดส่วนนี้ข้อมูลจะเป็นเพียงสัญญาณที่ประมวลผลไม่ได้//

**โครงสร้างการเชื่อมต่อเครือข่าย (Network Topology)**
//การวางรูปแบบทางกายภาพของเครือข่ายส่งผลโดยตรงต่อประสิทธิภาพและการสำรองข้อมูลในกรณีที่เกิดความผิดพลาด
1 Mesh Topology การเชื่อมต่อแบบตาข่ายที่ทุกอุปกรณ์เชื่อมถึงกันโดยตรง ให้ความทนทานสูงสุดเพราะมีเส้นทางสำรองข้อมูลตลอดเวลา แต่แลกมาด้วยค่าใช้จ่ายในการติดตั้งที่สูงมาก
2 Star Topology รูปแบบที่นิยมที่สุดในปัจจุบัน โดยทุกเครื่องจะเชื่อมเข้าหาจุดศูนย์กลางอย่าง Switch หากเครื่องใดเครื่องหนึ่งเสียหายจะไม่มีผลกระทบต่อเครื่องอื่น แต่หากจุดกลางเสียระบบจะล่มทั้งหมด
3 Bus Topology การใช้สายแกนหลักเพียงสายเดียวในการรับส่งข้อมูล มีข้อดีคือความประหยัดและติดตั้งง่าย แต่อาจเกิดปัญหาข้อมูลชนกันได้ง่ายหากมีปริมาณการใช้งานหนาแน่นและไม่เหมาะกับเครือข่ายขนาดใหญ่//

บทที่ 2: Physical Layer (ชั้นกายภาพ - รากฐานของบิตและสัญญาณ)
**การส่งผ่านข้อมูล: Digital vs. Analog**
//ข้อมูลในรูปแบบบิต (0 และ 1) จะถูกส่งผ่านสื่อกลางในรูปแบบของสัญญาณไฟฟ้าหรือคลื่นวิทยุที่แตกต่างกันตามวัตถุประสงค์การใช้งาน:
1 Digital Signal สัญญาณแบบขั้นบันไดที่มีค่าแน่นอนชัดเจน (เช่น 0V หรือ 5V) มีความแม่นยำสูงและทนต่อสัญญาณรบกวนได้ดีเยี่ยมในการสื่อสารระยะใกล้
2 Analog Signal สัญญาณรูปแบบคลื่นต่อเนื่องที่สามารถส่งไปได้ไกลกว่ามากผ่านเทคนิคการผสมสัญญาณ (Modulation) เช่น สัญญาณวิทยุหรือเสียงพื้นฐาน//

**สื่อนำข้อมูล (Transmission Media)**
//การเลือกสื่อกลางมีผลต่อปริมาณข้อมูล (Bandwidth) และความเร็วในการตอบสนอง (Latency) อย่างมีนัยสำคัญ:
1 Twisted Pair (สายคู่ตีเกลียว) สาย LAN ที่เราคุ้นเคย ใช้การตีเกลียวเพื่อลดการรบกวนของสนามแม่เหล็กไฟฟ้า (Crosstalk) เหมาะสำหรับการเชื่อมต่อภายในอาคาร
2 Coaxial Cable สายแกนทองแดงที่มีฉนวนหุ้มหนาแน่น มีความทนทานสูง ปัจจุบันยังพบในระบบการส่งสัญญาณโทรทัศน์หรืออินเทอร์เน็ตผ่านเคเบิล
3 Fiber Optics สื่อกลางที่ทรงพลังที่สุดด้วยการส่งข้อมูลผ่านลำแสง ข้อดีคือรองรับ Bandwidth มหาศาล ส่งได้ระยะไกลหลายกิโลเมตรโดยสัญญาณไม่ตก และไม่ถูกรบกวนจากคลื่นแม่เหล็กไฟฟ้า (EMI) เลย//

**แบนด์วิดท์ (Bandwidth) และทรูพุต (Throughput)**
//ในการวัดประสิทธิภาพการเชื่อมต่อ เราต้องแยกให้ออกระหว่าง "ขีดจำกัด" และ "การใช้งานจริง":
1 Bandwidth คือความกว้างสูงสุดของเส้นทางสื่อสารที่สามารถรองรับปริมาณข้อมูลได้ เปรียบเสมือนความกว้างของถนน
2 Throughput คือปริมาณข้อมูลที่ส่งผ่านไปได้จริงในเวลาหนึ่ง เปรียบเสมือนปริมาณรถที่วิ่งจริงบนถนน ซึ่งมักน้อยกว่า Bandwidth เนื่องจากปัจจัยรบกวน (Noise) หรือความแออัดของเครือข่ายในขณะนั้น//

บทที่ 3: Data Link Layer (ชั้นเชื่อมโยงข้อมูล - ผู้ดูแลความน่าเชื่อถือของการรับส่ง)
**การจัดรูปแบบข้อมูล (Framing)**
//บิตข้อมูลจะถูกนำมาห่อหุ้มด้วยโครงสร้างที่ชัดเจน เพื่อให้ผู้รับรู้ขอบเขตของข้อมูลแต่ละชุดประกอบด้วย:
Header ส่วนหน้ากระดาษที่เก็บข้อมูลสำคัญอย่าง MAC Address ของผู้ส่งและผู้รับ เพื่อระบุตัวตนบนเครือข่ายท้องถิ่น (LAN)
Payload ข้อมูลส่วนกลางที่เป็นเนื้อหาหลักซึ่งถูกส่งต่อมาจากเลเยอร์ที่สูงกว่า
Trailer ส่วนท้ายที่ใช้สำหรับบรรจุรหัสตรวจสอบความถูกต้อง เพื่อยืนยันว่าข้อมูลที่เดินทางมานั้นไม่ถูกบิดเบือนหรือเสียหายระหว่างทาง//

**การควบคุมการไหลของข้อมูล (Flow Control)**
//ชั้นนี้มีหน้าที่เป็น "จราจร" คอยกำกับปริมาณข้อมูลไม่ให้ผู้ส่งส่งเร็วเกินกว่าที่ผู้รับจะรับไหว เพื่อป้องกันปัญหาข้อมูลล้น (Buffer Overflow):
Stop-and-Wait กลไกพื้นฐานที่ผู้ส่งจะส่งเพียง 1 เฟรมแล้วหยุดรอสัญญาณตอบรับ (ACK) จากผู้รับก่อนส่งเฟรมถัดไป
Sliding Window กลไกขั้นสูงที่อนุญาตให้ส่งข้อมูลได้ทีละหลายเฟรมก่อนจะรอการยืนยัน ช่วยลดความหน่วงและเพิ่มประสิทธิภาพการใช้ช่องสัญญาณ (Throughput) ให้สูงขึ้นอย่างมหาศาล//

**การตรวจจับและแก้ไขความผิดพลาด (Error Control)**
//เมื่อข้อมูลเดินทางผ่านสื่อกลางที่มีสัญญาณรบกวน บิตข้อมูลอาจเกิดความผิดพลาดได้ ชั้น Data Link จึงมีระบบรักษาความปลอดภัยของข้อมูล:
Cyclic Redundancy Check (CRC) อัลกอริทึมทางคณิตศาสตร์ที่คำนวณค่าตรวจสอบจากข้อมูลทั้งหมดในเฟรม หากผู้รับคำนวณแล้วได้ค่าไม่ตรงกับที่แนบมาใน Trailer ระบบจะทำการปฏิเสธข้อมูลและขอให้ส่งใหม่ (Retransmission) ทันที//

**เลเยอร์ย่อย (Sub-layers) ของ Data Link**
//เพื่อให้เกิดความยืดหยุ่นในการทำงานร่วมกับฮาร์ดแวร์ที่หลากหลาย ชั้นนี้จึงถูกแบ่งหน้าที่ออกเป็น 2 ส่วนย่อย:
Logical Link Control (LLC) ทำหน้าที่ประสานงานกับชั้นเครือข่าย (Network Layer) เพื่อจัดการการไหลของข้อมูลและการควบคุมข้อผิดพลาดในระดับซอฟต์แวร์
Media Access Control (MAC) ส่วนที่ใกล้ชิดกับฮาร์ดแวร์ที่สุด ทำหน้าที่จัดการการเข้าถึงสื่อกลางว่าใครจะได้สิทธิ์ส่งข้อมูลก่อน-หลัง และใช้ MAC Address ซึ่งเป็นเลขประจำตัวที่ไม่ซ้ำกันของอุปกรณ์เน็ตเวิร์กทั่วโลกในการระบุเป้าหมาย//

บทที่ 4: MAC & Switching (การจัดการเครือข่ายท้องถิ่น - จากการชนกันสู่การส่งต่อที่ชาญฉลาด)
**MAC Address: ที่อยู่หนึ่งเดียวในโลก**
//อุปกรณ์เน็ตเวิร์กทุกชิ้นจะถูกประทับตราด้วยเลขประจำตัวที่ไม่ซ้ำกันเลยทั่วโลก เพื่อใช้ระบุตัวตนในระดับฮาร์ดแวร์:
โครงสร้าง เป็นเลขฐานสิบหกขนาด 48 บิต (เช่น 00:1A:2B:3C:4D:5E) ซึ่งถูกบันทึกไว้ในชิปเน็ตเวิร์กตั้งแต่โรงงาน
หน้าที่ ใช้ระบุตัวตนของอุปกรณ์ในระดับ Hardware (Physical Address) เพื่อให้การส่งข้อมูลภายในเครือข่ายท้องถิ่นส่งไปยังเครื่องเป้าหมายได้อย่างถูกต้อง//

**CSMA/CD: กฎจราจรของ Ethernet แบบดั้งเดิม**
//ในอดีตที่คอมพิวเตอร์ต้องแย่งกันใช้สายสัญญาณเส้นเดียว ระบบต้องมีกฎเพื่อป้องกันการ "พูดแทรก" หรือข้อมูลชนกัน:
Carrier Sense ก่อนเริ่มส่งข้อมูล อุปกรณ์ต้อง "แอบฟัง" สัญญาณในสายก่อนว่าว่างหรือไม่
Multiple Access ทุกเครื่องในเครือข่ายมีสิทธิ์เข้าถึงสื่อกลางเท่าเทียมกัน
Collision Detection หากมีการส่งพร้อมกันจนข้อมูลชนกัน ระบบจะสั่งหยุดส่งทันที แล้วสุ่มรอเวลา (Exponential Backoff) ก่อนจะเริ่มกระบวนการส่งใหม่อีกครั้ง//

**วิวัฒนาการจาก Hub สู่ Switch**
//อุปกรณ์ที่ใช้เชื่อมต่อส่งผลโดยตรงต่อค่า Latency และประสิทธิภาพรวมของเครือข่าย:
Hub (Layer 1) เป็นอุปกรณ์แบบ "กระจายเสียง" เมื่อได้รับข้อมูลจะส่งออกไปทุกพอร์ต (Broadcast) ทำให้เกิดปัญหาข้อมูลชนกันสูงและสิ้นเปลืองแบนด์วิดท์
Switch (Layer 2) อุปกรณ์ที่ชาญฉลาดกว่าด้วยระบบ MAC Learning ซึ่งจะจดจำว่าใครอยู่ที่พอร์ตไหนลงในตาราง (MAC Address Table) ทำให้ส่งข้อมูลได้แบบ Unicast (ส่งตรงถึงผู้รับ) และทำงานแบบ Full-Duplex คือรับและส่งได้ในเวลาเดียวกัน//

**VLAN (Virtual LAN) การแบ่งโซนเสมือน**
//เทคโนโลยีที่ช่วยให้เราบริหารจัดการเครือข่ายขนาดใหญ่ให้เป็นสัดส่วนได้ง่ายขึ้นโดยไม่ต้องซื้ออุปกรณ์เพิ่ม
ความเป็นส่วนตัว การแบ่งกลุ่มเครือข่ายเสมือนเพื่อความปลอดภัย เช่น การแยกวงเครือข่ายของระบบหลังบ้านออกจากส่วนที่นักศึกษาใช้งานทั่วไป
ลด Broadcast Traffic ป้องกันไม่ให้สัญญาณรบกวนหรือการประกาศข้อมูลจากเครื่องหนึ่งกระจายไปรบกวนเครื่องในกลุ่มอื่น ช่วยให้เครือข่ายมีความลื่นไหลและเสถียรยิ่งขึ้น//

บทที่ 5: Network Layer (ชั้นเครือข่าย - ระบบนำทางและการกำหนดที่อยู่ทั่วโลก)
**IP Address บ้านเลขที่บนโลกไซเบอร์**
//เพื่อให้ข้อมูลส่งไปถูกที่ ทุกอุปกรณ์ที่เชื่อมต่ออินเทอร์เน็ตต้องมีที่อยู่ระบุตัวตนในระดับซอฟต์แวร์ (Logical Address) ซึ่งแบ่งเป็น 2 มาตรฐานหลัก:
1 IPv4 (Internet Protocol version 4): รูปแบบเลข 32 บิต แบ่งเป็น 4 ชุด (เช่น 192.168.1.1) มีข้อจำกัดด้านจำนวนซึ่งปัจจุบันไม่เพียงพอต่อความต้องการ จึงต้องมีเทคนิคอย่าง NAT เข้ามาช่วยจัดการ
2 IPv6 (Internet Protocol version 6): มาตรฐานใหม่ในรูปแบบเลขฐานสิบหก 128 บิต มีจำนวนมหาศาลพอที่จะรองรับอุปกรณ์ IoT และสมาร์ทดีไวซ์ในอนาคตได้อย่างไร้ขีดจำกัดโดยไม่ต้องกังวลเรื่องไอพีเต็มอีกต่อไป//

**Subnetting การจัดสรรพื้นที่อย่างชาญฉลาด**
//การบริหารจัดการไอพีให้คุ้มค่าและปลอดภัยเป็นทักษะสำคัญสำหรับนักพัฒนาและผู้ดูแลระบบเครือข่าย:
1 การแบ่งเครือข่ายย่อย: คือการหั่นเครือข่ายขนาดใหญ่หนึ่งเครือข่ายออกเป็นหลายเครือข่ายย่อย (Subnets) เพื่อลดความแออัดของข้อมูลและเพิ่มความปลอดภัยในการเข้าถึงข้อมูล
2 ประโยชน์เชิงโครงสร้าง: ช่วยให้สามารถจัดการกลุ่มไอพีให้เป็นสัดส่วน เช่น การแยกวงไอพีของเซิร์ฟเวอร์หลัก ออกจากเครือข่าย Wi-Fi สำหรับนักศึกษาหรือเครื่องคอมพิวเตอร์ในแล็บปฏิบัติการ//

**เราเตอร์ (Router) และการเลือกเส้นทาง (Routing)**
//ถ้า Switch คือคนคุมจราจรในซอย Router ก็คือเจ้าหน้าที่วางแผนเส้นทางบนไฮเวย์ที่เชื่อมโยงเครือข่ายต่างๆ เข้าด้วยกัน:
1 Routing Table: เราเตอร์จะจัดเก็บตารางเส้นทางเพื่อวิเคราะห์ว่าแพ็กเก็ตที่ได้รับมาควรจะส่งออกไปยังพอร์ตใด หรือผ่านเราเตอร์ตัวถัดไป (Next Hop) ตัวใดเพื่อให้ถึงเป้าหมายเร็วที่สุด
2 Routing Protocols: โปรโตคอลอย่าง OSPF หรือ BGP คือภาษาที่เราเตอร์ใช้คุยกันเพื่ออัปเดตข้อมูลและหาทางลัดที่สั้นที่สุด รวมถึงหลีกเลี่ยงเส้นทางที่ขัดข้องหรือล่มสลาย//

**ICMP หน่วยกู้ภัยประจำเครือข่าย**
//เวลาเกิดปัญหาการเชื่อมต่อหรืออาการแล็กในระบบ โปรโตคอล ICMP จะเป็นเครื่องมือแรกในการวินิจฉัยปัญหา:
1 Ping การส่งแพ็กเก็ตขนาดเล็กไปตรวจสอบสถานะปลายทางว่ายังเชื่อมต่ออยู่หรือไม่ พร้อมวัดเวลาในการเดินทาง (Round Trip Time - RTT) เพื่อประเมินความเร็ว
2 Traceroute: เครื่องมือตรวจสอบเส้นทางว่าแพ็กเก็ตวิ่งผ่านเราเตอร์ตัวใดบ้าง ช่วยระบุจุดที่เกิดความหน่วงหรือจุดที่ข้อมูลขาดหายได้อย่างแม่นยำ//

บทที่ 6: Transport Layer (ชั้นขนส่ง - ผู้จัดการการส่งมอบข้อมูลถึงมือแอปพลิเคชัน)
**ชั้นขนส่ง (Transport Layer)**
//ในขณะที่ Network Layer พาข้อมูลข้ามโลกมาจนถึงหน้าบ้าน (IP Address) Transport Layer (Layer 4) คือคนที่จะพาข้อมูลขึ้นไปส่งให้ถึงห้องที่ถูกต้องภายในบ้านครับ ชั้นนี้ทำหน้าที่ดูแลการสื่อสารระหว่างแอปพลิเคชัน (Process-to-Process) ให้มีความเสถียรและไปถูกที่ผ่านการระบุ "Port Number" ครับ//

**Port Number (ที่อยู่ของแอปพลิเคชัน)**
//ในคอมพิวเตอร์หนึ่งเครื่องอาจจะรันหลายแอปฯ พร้อมกัน ข้อมูลที่เข้ามาในไอพีเดียวกันจะแยกย้ายไปหาแอปฯ ถูกได้อย่างไร? คำตอบคือ Port Number ครับ
Well-known Ports (0-1023) พอร์ตมาตรฐาน เช่น HTTP (80), HTTPS (443), SSH (22)
Registered/Dynamic Ports พอร์ตที่แอปฯ ทั่วไปเลือกใช้ เช่น FastAPI (8000) หรือ Next.js (3000)//

**TCP vs. UDP (เลือกความชัวร์ หรือ เลือกความเร็ว?)**
//นี่คือหัวใจของชั้นนี้เลยครับ ในฐานะนักพัฒนาและเกมเมอร์ คุณต้องเลือกเครื่องมือให้เหมาะกับงาน//

**TCP (Transmission Control Protocol) - "เน้นความชัวร์"**
//Connection-Oriented ต้องทำ Three-way Handshake เพื่อทำความรู้จักกันก่อนส่งข้อมูล มี Reliability ในการตรวจสอบว่าข้อมูลครบไหม ถ้าหายจะส่งใหม่ และมี Flow & Congestion Control ปรับความเร็วการส่งตามสภาพเครือข่าย ไม่ให้ผู้รับ "สำลัก" ข้อมูล
เหมาะสำหรับ เว็บไซต์ (HTTP), ส่งไฟล์ (FTP), อีเมล (SMTP) ซึ่งข้อมูลขาดแม้แต่บิตเดียวไม่ได้//

**UDP (User Datagram Protocol) - "เน้นความเร็ว"**
//ตามภาพ image_4a2011.png สำหรับโปรโตคอลที่เน้นความเร็วอย่าง UDP จะเป็นแบบ Connectionless อยากส่งก็ส่งเลย ไม่มีการทักทาย ไม่มีการรอตอบรับ (No ACK) มี Low Overhead เพราะแพ็กเก็ตมีขนาดเล็กมาก และให้ Speed ที่รวดเร็วที่สุดเพราะไม่ต้องรอตรวจสอบ
เหมาะสำหรับ การเล่นเกมออนไลน์ (CS2), การสตรีมมิ่งวิดีโอ (YouTube Live), ระบบ VoIP (Discord) ซึ่งถ้าข้อมูลหายไปบางเฟรมก็แค่ "กระตุก" นิดหน่อย แต่ความลื่นไหลต้องมาก่อน//

บทที่ 7: Application Protocol (ชั้นประยุกต์ - ประตูเชื่อมต่อระหว่างผู้ใช้และเครือข่าย)
**ชั้นการประยุกต์ใช้ (Application Layer)**
//ในที่สุดเราก็เดินทางมาถึง Layer 7 ซึ่งเป็นชั้นที่อยู่ใกล้ตัวคุณมากที่สุดครับ ชั้นนี้ไม่ได้ทำหน้าที่ส่งบิตหรือหาเส้นทางเหมือนเลเยอร์ล่างๆ แต่หน้าที่ของมันคือการจัดการ "ความหมาย" ของข้อมูลเพื่อให้ซอฟต์แวร์ เช่น เว็บเบราว์เซอร์หรือแอปพลิเคชันมือถือ สามารถทำงานร่วมกับเครือข่ายได้อย่างสมบูรณ์ครับ//

**สถาปัตยกรรม Client-Server (หัวใจของเว็บแอปฯ)**
//ก่อนจะคุยเรื่องโปรโตคอล เราต้องเข้าใจโมเดลหลักที่ใช้ในปัจจุบันก่อนครับ
Client (ลูกข่าย) อุปกรณ์ของคุณ เช่น Browser หรือแอปฯ CSLearning ที่ทำหน้าที่ส่งคำขอ (Request)
Server (เซิร์ฟเวอร์) เครื่องคอมพิวเตอร์ที่เก็บข้อมูลและรอประมวลผลคำขอ เพื่อส่งข้อมูลกลับมา (Response)//

**โปรโตคอลที่สำคัญในชีวิตประจำวัน**
//ในฐานะนักพัฒนาสาย Full-stack โปรโตคอลเหล่านี้คือสิ่งที่คุณต้อง "เขียน" และ "จัดการ" อยู่ทุกวันครับ
HTTP/HTTPS (HyperText Transfer Protocol) โปรโตคอลหลักในการรับส่งข้อมูลบนเว็บ ปัจจุบันต้องเป็น HTTPS (Secure) เสมอเพื่อการเข้ารหัสข้อมูลผ่าน TLS/SSL
DNS (Domain Name System) เปรียบเสมือนสมุดโทรศัพท์ของโลกอินเทอร์เน็ต ทำหน้าที่แปลงชื่อเว็บไซต์ เช่น github.com ให้กลายเป็น IP Address ที่เราเตอร์เข้าใจ
DHCP (Dynamic Host Configuration Protocol) ตัวช่วยที่ทำให้คุณไม่ต้องตั้งค่า IP เองทุกครั้งที่ต่อ Wi-Fi มหาวิทยาลัย ระบบจะแจกจ่ายไอพีให้คุณโดยอัตโนมัติ
SMTP / POP3 / IMAP ชุดโปรโตคอลสำหรับการส่งและรับอีเมล
FTP (File Transfer Protocol) ใช้สำหรับการรับส่งไฟล์ขนาดใหญ่ระหว่างเครื่อง//

บทที่ 8: Network Management (การบริหารจัดการเครือข่าย - การดูแลรักษา ความเสถียร และความปลอดภัย)
**SNMP (ดวงตาของนักบริหารเครือข่าย)**
//ในการดูแลเครือข่ายขนาดใหญ่ เราไม่สามารถเดินไปเช็คอุปกรณ์ทีละตัวได้ โปรโตคอล SNMP (Simple Network Management Protocol) จึงถูกสร้างขึ้นเพื่อให้เราเฝ้าดู (Monitor) และจัดการอุปกรณ์จากส่วนกลางได้
Managed Devices อุปกรณ์ต่างๆ เช่น Router, Switch, Server
Agent ซอฟต์แวร์จิ๋วที่อยู่ในอุปกรณ์ คอยเก็บข้อมูล เช่น อุณหภูมิ CPU หรือปริมาณ Bandwidth ที่ใช้
NMS (Network Management Station) เครื่องคอมพิวเตอร์ส่วนกลางที่คอยรวบรวมข้อมูลจาก Agent มาแสดงผลเป็นกราฟหรือแจ้งเตือนเมื่อเกิดความผิดปกติ//

**Network Security (เกราะป้องกัน 3 ชั้น)**
//ความปลอดภัยคือหัวใจหลักที่ทำให้ระบบน่าเชื่อถือ โดยมีเครื่องมือสำคัญดังนี้ครับ
Firewall (กำแพงไฟ) ทำหน้าที่เป็นด่านตรวจคนเข้าเมือง คอยคัดกรองแพ็กเก็ตข้อมูลตามกฎที่ตั้งไว้ เช่น อนุญาตให้เฉพาะพอร์ต 443 เข้ามาได้ แต่บล็อกพอร์ตอื่นๆ
IDS/IPS (ระบบตรวจจับและป้องกันการบุกรุก) เปรียบเสมือนกล้องวงจรปิดที่คอยวิเคราะห์พฤติกรรมที่ผิดปกติ เช่น การพยายามสุ่มรหัสผ่านรัวๆ และทำการแจ้งเตือนหรือตัดการเชื่อมต่อทันที
VPN (Virtual Private Network) การสร้าง "อุโมงค์ลับ" ที่มีการเข้ารหัสข้อมูล (Encryption) ช่วยให้รับส่งข้อมูลสำคัญได้อย่างปลอดภัยเสมือนนั่งอยู่ในวง LAN เดียวกัน//

**การจัดการปัญหาคอขวด (Bottleneck Management)**
//ปัญหาคอขวดคือจุดที่ทำให้ประสิทธิภาพของเครือข่ายลดลง ซึ่งส่งผลต่อ Latency โดยตรง
จุดที่มักเกิดปัญหา เราเตอร์ประมวลผลไม่ทัน, สายสัญญาณหน่วงเกินไป (Throughput ต่ำ), หรือเซิร์ฟเวอร์โดน Request ถล่มจนรับไม่ไหว
วิธีแก้ การทำ Load Balancing (กระจายงานให้เซิร์ฟเวอร์หลายตัว) และการใช้ QoS (Quality of Service) เพื่อให้ความสำคัญกับข้อมูลที่อ่อนไหวต่อเวลา เช่น ข้อมูลเกม หรือ Voice Chat ให้วิ่งผ่านไปก่อนข้อมูลการโหลดไฟล์ทั่วไป//
