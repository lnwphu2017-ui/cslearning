3. Data Structures
บทที่ 1 รายการเชื่อมโยง (Linked Lists)
**กายวิภาคของโหนด (Anatomy of a Node)** //ใน Linked List เราไม่เก็บข้อมูลลงในช่องดัชนี (Index) เหมือน Array แต่เราเก็บไว้ในสิ่งที่เรียกว่า Node ซึ่งประกอบด้วยสองส่วนสำคัญData ข้อมูลที่เราต้องการจัดเก็บ เช่น เลข ข้อความ หรือ ObjectNext (Pointer) ตัวชี้ที่เก็บพิกัดหน่วยความจำของโหนดถัดไป//

**ประเภทของ Linked Lists** //เราสามารถเชื่อมโยงโหนดเหล่านี้ได้หลายรูปแบบ ตามความต้องการใช้งานSingly Linked List ชี้ไปข้างหน้าทางเดียว ประหยัดหน่วยความจำแต่ย้อนกลับไม่ได้ ต้องเริ่มหาจากหัวแถวเสมอDoubly Linked List ชี้ทั้งไปข้างหน้าและย้อนกลับ เดินหน้า-ถอยหลังได้อิสระ ลบโหนดได้ง่ายกว่า แต่ใช้หน่วยความจำเพิ่มขึ้นCircular Linked List โหนดสุดท้ายชี้กลับมาที่โหนดแรก วนลูปข้อมูลได้ไม่รู้จบ เหมาะกับงานคิว ถ้าจัดการไม่ดีอาจเกิด Infinite Loop ได้//

*ทำไมต้อง Linked List? (Performance Insight)**//จุดเด่นที่สุดที่ทำให้ Linked List ชนะ Array ขาดลอยคือการ Insert/Delete ตรงกลางข้อมูลใน Array หากคุณต้องการแทรกข้อมูลที่ตำแหน่งที่ 2 คุณต้องขยับข้อมูลที่ตำแหน่ง 2 ถึง $n$ ไปทางขวาทั้งหมด ($O(n)$)ใน Linked List คุณแค่เดินไปที่โหนดนั้น แล้วสั่งให้โหนดก่อนหน้าเลิกชี้คนเดิมแล้วมาชี้คนใหม่ และให้คนใหม่ชี้ไปที่คนเดิมแทน กระบวนการนี้เกิดขึ้นทันทีด้วยความเร็วคงที่ ($O(1)$) หากเราอยู่ที่ตำแหน่งนั้นแล้ว//

บทที่ 2: Stacks & Queues (ระเบียบการเข้า-ออกของข้อมูล)
**Stack: เข้าทีหลัง ออกก่อน (Last-In, First-Out - LIFO)**
//นึกถึงการ "ซ้อนจาน" หรือกระบอกใส่เทนนิสครับ จานใบสุดท้ายที่คุณวางลงบนตั้ง จะเป็นใบแรกที่คุณต้องหยิบออกเสมอ โครงสร้างนี้จึงมีความเรียบง่ายแต่ทรงพลังมากในโลกคอมพิวเตอร์
ปฏิบัติการหลัก ได้แก่ Push การเพิ่มข้อมูลลงไปที่ส่วนบนสุด (Top), Pop การดึงข้อมูลออกจากส่วนบนสุดเท่านั้น และ Peek การแอบดูข้อมูลที่อยู่บนสุดโดยไม่ดึงออก
ตัวอย่างการใช้งานจริง เช่น Call Stack ในการจำจุดทำงานของฟังก์ชัน และ Undo Mechanism สำหรับการกดย้อนกลับสถานะล่าสุดด้วย Ctrl+Z//

**Queue: มาก่อน ได้ก่อน (First-In, First-Out - FIFO)**
//หาก Stack คือการซ้อนจาน Queue ก็คือการ "เข้าแถวซื้อกาแฟ" ครับ ใครที่มาถึงคิวเป็นคนแรก ย่อมได้รับบริการก่อนและออกจากแถวเป็นคนแรก โครงสร้างนี้ใช้จัดการลำดับความสำคัญตามเวลา
ปฏิบัติการหลัก ได้แก่ Enqueue การต่อแถวที่ส่วนท้าย (Rear) และ Dequeue การออกจากแถวที่ส่วนหน้า (Front)
ตัวอย่างการใช้งานจริง เช่น Task Scheduling ในระบบปฏิบัติการเพื่อจัดคิวการทำงานของ CPU และ Message Broker ที่ใช้ส่งข้อมูลระหว่าง Server ในระบบขนาดใหญ่//

บทที่ 3: Binary Trees (โครงสร้างข้อมูลแบบลำดับชั้น)
**กายวิภาคของต้นไม้ทวิภาค**//ใน Binary Tree แต่ละ Node จะมีกฎเหล็กว่า "ห้ามมีลูกเกิน 2 คน" คือซ้ายและขวา ซึ่งประกอบด้วยส่วนประกอบสำคัญที่ทำให้โครงสร้างมีความสัมพันธ์แบบเป็นชั้นๆRoot (ราก) โหนดที่อยู่บนสุด เป็นจุดเริ่มต้นของต้นไม้Parent (โหนดพ่อแม่) โหนดที่มีกิ่งก้านแยกออกมาChild (โหนดลูก) โหนดที่ถูกชี้มาจากด้านบน แบ่งเป็น Left Child และ Right ChildLeaf (ใบ) โหนดที่ไม่มีลูกแล้ว อยู่ปลายสุดของกิ่ง//

**ศิลปะการท่องไปในต้นไม้ (Tree Traversal)**//การจะอ่านข้อมูลในต้นไม้ออกมาให้ครบทุกโหนด ต้องใช้วิธีการเรียกใช้ซ้ำ (Recursion) ในการเดินไปตามกิ่งก้าน ซึ่งมี 3 รูปแบบมาตรฐานที่ให้ผลลัพธ์ต่างกันPreorder (Root -> Left -> Right) ใช้ในการคัดลอกต้นไม้ หรือสร้างนิพจน์คณิตศาสตร์แบบ PrefixInorder (Left -> Root -> Right) ยอดนิยมที่สุด เพราะถ้าเป็น Search Tree ข้อมูลจะถูกเรียงลำดับจากน้อยไปมากทันทีPostorder (Left -> Right -> Root) ใช้ในการลบโหนดเพื่อเคลียร์ลูกก่อนลบพ่อแม่ หรือใช้คำนวณพื้นที่ในระบบไฟล์//

**ทำไมต้องเป็น Binary Tree?**//ต้นไม้เป็นทางเลือกที่ยอดเยี่ยมในการจัดการข้อมูลให้เข้าถึงได้รวดเร็วขึ้นกว่าการเก็บข้อมูลแบบเส้นตรงEfficiency ในต้นไม้ที่สมดุลการค้นหาข้อมูลจะใช้เวลาเพียง O(\log n) ซึ่งเร็วกว่าการไล่หาใน Array ที่ใช้ O(n) มหาศาล เช่น ข้อมูล 1,000,000 ตัว อาจหาเพียงแค่ 20 ครั้งก็เจอOrganization ระบบไฟล์ในเครื่องคอมพิวเตอร์อย่าง Folders และ Sub-folders หรือโครงสร้าง HTML (DOM) ต่างใช้พื้นฐานมาจาก Tree ทั้งสิ้น//

บทที่ 4: Balanced Trees (ต้นไม้สมดุล - เกราะป้องกันประสิทธิภาพ)
**AVL Tree: ความสมดุลที่เข้มงวด**
//AVL Tree เป็นต้นไม้สมดุลประเภทแรกที่ถูกคิดค้นขึ้นโดยใช้หลักการของ Balance Factor (BF) เพื่อป้องกันไม่ให้เกิดอาการ Skewed Tree ที่ทำให้ประสิทธิภาพการทำงานลดลง
กฎเหล็ก ความสูงของต้นไม้ฝั่งซ้ายและฝั่งขวาของทุกโหนด ต้องต่างกันไม่เกิน 1 หากมีการเพิ่มข้อมูลแล้วทำให้ BF กลายเป็น 2 หรือ -2 ระบบจะทำการ Rotation ทันที
การจัดการ ระบบจะทำการหมุนโหนดเพื่อดึงกิ่งที่ยาวเกินไปให้กลับมาสมดุลเสมอ
ข้อดี ค้นหาได้เร็วมากเพราะต้นไม้มีความแน่นและสมมาตรที่สุด//

**Red-Black Tree: ความยืดหยุ่นในโลกใช้งานจริง**
//โครงสร้างที่ถูกนำไปใช้ใน Library มาตรฐานของหลายภาษาและใน Linux Kernel เนื่องจากให้ประสิทธิภาพที่สมดุลระหว่างการค้นหาและการแก้ไขข้อมูล
หลักการ ใช้การพ่นสีโหนดเป็นสีแดงหรือดำ และใช้กฎ 5 ข้อในการควบคุม เช่น Root ต้องเป็นสีดำ และโหนดสีแดงห้ามอยู่ติดกัน
ความแตกต่าง ยอมให้ต้นไม้เอียงได้เล็กน้อยและไม่เข้มงวดเท่า AVL ทำให้เมื่อมีการเพิ่มหรือลบข้อมูลไม่ต้องหมุนโหนดบ่อยครั้ง
ข้อดี เหมาะสำหรับระบบที่ต้องมีการเพิ่มและลบข้อมูลบ่อยๆ โดยยังคงความเร็วในการค้นหาที่น่าพอใจ//

**พลังของการหมุนโหนด (Tree Rotations)**
//การหมุนโหนดคือเวทมนตร์ที่เปลี่ยนโครงสร้างต้นไม้โดยที่ลำดับของข้อมูลยังถูกต้องและค่า Inorder ไม่เปลี่ยนแปลง ซึ่งเป็นกลไกสำคัญของต้นไม้สมดุล
Single Rotation ได้แก่ การหมุนซ้าย (Left Rotation) หรือ การหมุนขวา (Right Rotation)
Double Rotation ได้แก่ การหมุนซ้าย-ขวา (Left-Right Rotation) หรือ การหมุนขวา-ซ้าย (Right-Left Rotation)//

บทที่ 5: Binary Heaps (หัวใจของคิวลำดับความสำคัญ)
**Binary Heap คืออะไร?**//Binary Heap คือต้นไม้ทวิภาคประเภทหนึ่งที่มีความพิเศษที่เน้นการจัดลำดับความสำคัญของข้อมูลมากกว่าการเรียงลำดับจากซ้ายไปขวาเหมือน BST ทั่วไปShape Property (รูปร่าง) ต้องเป็น Complete Binary Tree เสมอ โดยข้อมูลต้องเติมให้เต็มทีละชั้นจากซ้ายไปขวาโดยห้ามมีรูโหว่Heap Property (ระเบียบ) แบ่งเป็น Max-Heap ที่โหนดพ่อแม่ต้องมีค่ามากกว่าหรือเท่ากับลูกเสมอ และ Min-Heap ที่โหนดพ่อแม่ต้องมีค่าน้อยกว่าหรือเท่ากับลูกเสมอ ทำให้ค่าที่สำคัญที่สุดอยู่ที่ Root เสมอ//

**การจัดเก็บข้อมูลที่ชาญฉลา Array Representation**//สิ่งที่ทำให้ Heap ทรงพลังคือเราสามารถเก็บข้อมูลลงใน Array ได้โดยตรงโดยไม่ต้องใช้ Pointer ทำให้ประหยัดหน่วยความจำและช่วยให้ CPU Cache ทำงานได้เร็วขึ้นการคำนวณดัชนี หากโหนดพ่อแม่อยู่ที่ดัชนี $i$ จะสามารถหาลูกซ้ายได้ที่ $2i + 1$ ลูกขวาที่ $2i + 2$ และหาพ่อแม่ได้ที่ $(i - 1) / 2$Performance Note การใช้ Array ช่วยเพิ่ม Cache Locality ซึ่งส่งผลโดยตรงต่อความเร็วในการประมวลผลของ CPU//

**ปฏิบัติการหลักของ Heap**//กระบวนการจัดการข้อมูลใน Heap ถูกออกแบบมาเพื่อรักษาโครงสร้างและความสมดุลให้เป็นไปตามกฎเสมอPeek ($O(1)$) การดูค่าที่อยู่บนยอดสุดหรือดัชนีที่ 0 ทำได้ทันทีเพราะเป็นจุดรวมของงานที่สำคัญที่สุดInsert ($O(\log n)$) การเพิ่มข้อมูลใหม่ไว้ท้ายสุดแล้วทำการ Sift Up หรือลอยขึ้นเพื่อจัดระเบียบใหม่ตามกฎ Heap PropertyExtract ($O(\log n)$) การดึงค่าบนยอดออกแล้วนำตัวสุดท้ายมาวางแทน จากนั้นทำการ Sift Down หรือ Heapify เพื่อจัดระเบียบให้ตัวที่เหมาะสมที่สุดขึ้นมาเป็น Root คนใหม่//

**การประยุกต์ใช้งานจริง**//โครงสร้างแบบ Heap ถูกนำไปใช้ในระบบที่ต้องการการตัดสินใจตามลำดับความสำคัญอย่างรวดเร็วOS Scheduling ใช้ Min-Heap ในการเลือก Process ที่มี Deadline ใกล้ที่สุดขึ้นมาประมวลผลก่อนDijkstra's Algorithm ใช้ในระบบนำทางอย่าง Google Maps เพื่อเลือกจุดที่สั้นที่สุดในการหาเส้นทางHeapsort อัลกอริทึมการเรียงลำดับที่มีประสิทธิภาพคงที่และไม่กินพื้นที่หน่วยความจำเพิ่มเติม//

บทที่ 6: Hashing (เวทมนตร์แห่งความเร็วระดับคงที่)
**Hash Function เครื่องจักรแปลงคีย์**//หัวใจของ Hashing คือสมการคณิตศาสตร์ที่รับข้อมูลขาเข้าแล้วประมวลผลออกมาเป็นตัวเลขดัชนีสำหรับเก็บข้อมูลในตาราง เพื่อให้เข้าถึงข้อมูลได้ด้วยความเร็วคงที่ในระดับ O(1)Deterministic ใส่คำเดิมต้องได้เลขเดิมเสมอเพื่อความแม่นยำในการเรียกคืนข้อมูลFast ต้องคำนวณได้รวดเร็วไม่กินทรัพยากร CPU จนกลายเป็นคอขวดของระบบUniformity ต้องกระจายข้อมูลให้ทั่วตารางเพื่อไม่ให้ไปกระจุกตัวอยู่ที่ดัชนีใดดัชนีหนึ่งเพียงอย่างเดียว//

**เมื่อ "การชน" เกิดขึ้น (Collision)**//เมื่อพื้นที่ในตารางมีจำกัดแต่คีย์มีปริมาณมหาศาล ทำให้คีย์ที่ต่างกันมีโอกาสถูกคำนวณออกมาได้ดัชนีเดียวกัน จึงต้องมีวิธีการจัดการที่มีประสิทธิภาพChaining (Separate Chaining) ใช้หลักการห้อยท้ายด้วยการสร้าง Linked List ต่อท้ายในช่องที่มีข้อมูลอยู่แล้ว ทำให้จัดการง่ายและข้อมูลล้นตารางได้Open Addressing ใช้หลักการหาบ้านใหม่โดยขยับไปหาช่องว่างถัดไป เช่น Linear Probing (ขยับทีละช่อง), Quadratic Probing (ขยับแบบยกกำลังสอง) หรือ Double Hashing (ใช้ฟังก์ชันที่สองคำนวณระยะกระโดด)//

**Load Factor และการรีดประสิทธิภาพ (Optimization)**//ประสิทธิภาพของ Hash Table ขึ้นอยู่กับอัตราส่วนระหว่างจำนวนข้อมูลกับขนาดตาราง ซึ่งส่งผลโดยตรงต่อโอกาสการเกิดการชนกันของข้อมูลLoad Factor (alpha) หากค่าเข้าใกล้ 0.7 - 0.8 ประสิทธิภาพจะลดลงอย่างเห็นได้ชัดเนื่องจากการชนกันที่ถี่ขึ้นRehashing วิธีแก้ปัญหาโดยการขยายขนาดตารางให้ใหญ่ขึ้นเป็นเท่าตัว แล้วนำข้อมูลเดิมทั้งหมดมาคำนวณตำแหน่งใหม่เพื่อให้การกระจายตัวกลับมาสมดุลอีกครั้ง//

**การประยุกต์ใช้งานในโลกจริง**//Hashing เป็นเทคโนโลยีเบื้องหลังที่สำคัญในระบบคอมพิวเตอร์ยุคใหม่ที่ต้องการความเร็วในการตอบสนองสูงสุดDatabase Indexing ใช้ในการค้นหา Primary Key เพื่อเข้าถึงเรคคอร์ดข้อมูลในฐานข้อมูลได้อย่างรวดเร็วCaching ระบบอย่าง Redis หรือ Memcached ใช้เพื่อดึงข้อมูลที่ใช้งานบ่อยออกมาแสดงผลได้ทันทีCryptography & Security ใช้ในการเก็บรหัสผ่านอย่างปลอดภัยหรือการทำ Checksum เพื่อตรวจสอบความถูกต้องของไฟล์Compilers ใช้สร้าง Symbol Table สำหรับจดจำชื่อตัวแปรและโครงสร้างโค้ดในระหว่างกระบวนการคอมไพล์โปรแกรม//

บทที่ 7: กราฟ (Graphs - โครงสร้างความสัมพันธ์ไร้ขีดจำกัด)
**กายวิภาคของกราฟ (Vertices & Edges)**
//กราฟประกอบด้วยส่วนประกอบหลักเพียง 2 อย่าง แต่สามารถสร้างโมเดลที่ซับซ้อนได้มหาศาลเพื่อสะท้อนความเป็นจริงของโลกที่ทุกอย่างเชื่อมโยงกันเป็นเครือข่าย
Vertex (Node) จุดยอดที่ใช้แทนวัตถุ เช่น เมือง ผู้ใช้งาน หรือคลังความรู้ในระบบ RAG
Edge เส้นเชื่อมที่ใช้แทนความสัมพันธ์ระหว่างวัตถุเหล่านั้น
Directed vs. Undirected เส้นเชื่อมแบบระบุทิศทาง เช่น การกด Follow กับแบบไม่ระบุทิศทาง เช่น การเป็นเพื่อนบน Facebook
Weighted vs. Unweighted เส้นเชื่อมที่มีค่าน้ำหนัก เช่น ระยะทางหรือค่า Latency กับแบบที่ทุกเส้นมีความสำคัญเท่ากัน//

**การทำให้คอมพิวเตอร์ "เห็น" กราฟ**
//คอมพิวเตอร์ไม่สามารถมองรูปวาดกราฟได้ เราจึงต้องแปลงให้อยู่ในรูปของโครงสร้างข้อมูลเพื่อให้ระบบประมวลผลความสัมพันธ์ได้
Adjacency Matrix ใช้ตาราง 2 มิติ (2D Array) ตรวจสอบความสัมพันธ์ระหว่าง 2 จุดได้ทันที เหมาะสำหรับกราฟที่มีเส้นเชื่อมหนาแน่น (Dense Graph)
Adjacency List ใช้ Array ของ Linked List ประหยัดพื้นที่หน่วยความจำมหาศาล เหมาะสำหรับกราฟที่มีเส้นเชื่อมเบาบาง (Sparse Graph)//

**อัลกอริทึมการท่องไปในกราฟ (Graph Traversal)**
//เมื่อมีโครงข่ายขนาดใหญ่ เราต้องมีกลยุทธ์ในการค้นหาข้อมูลข้างใน ซึ่งมี 2 รูปแบบระดับตำนาน
Breadth-First Search (BFS) เน้นกว้างโดยใช้ Queue เข้ามาช่วยสำรวจโหนดเพื่อนบ้านรอบตัวให้ครบก่อนขยับไปชั้นถัดไป จุดเด่นคือรับประกันการเจอเส้นทางที่สั้นที่สุดในกราฟที่ไม่มีค่าน้ำหนัก
Depth-First Search (DFS) เน้นลึกโดยใช้ Stack หรือ Recursion ดำดิ่งลงไปตามเส้นทางจนสุดก่อนย้อนกลับมาสำรวจเส้นทางอื่น เก่งในการตรวจหาวงจรและใช้หน่วยความจำน้อยกว่าในบางกรณี//

**การประยุกต์ใช้งานในเทคโนโลยียุคใหม่**
//โครงสร้างกราฟคือรากฐานสำคัญของเทคโนโลยีขั้นสูงที่ต้องจัดการความสัมพันธ์ที่ซับซ้อน
Knowledge Graphs ช่วยให้ระบบ AI เข้าใจความสัมพันธ์ของแนวคิดต่างๆ ในเชิงบริบทไม่ใช่แค่สถิติ
Network Routing ใช้ในการส่งข้อมูลในอินเทอร์เน็ตให้ถึงที่หมายเร็วที่สุดโดยเลี่ยงจุดที่การจราจรหนาแน่น
Game AI ใช้คำนวณ Pathfinding ให้ตัวละครเดินหลบสิ่งกีดขวางได้อย่างเป็นธรรมชาติ//

บทที่ 8: Advanced Sorting (การจัดเรียงข้อมูลระดับสูง)
**Merge Sort จอมวางแผนผู้ซื่อสัตย์**//Advanced Sorting ในกลุ่มนี้ใช้แนวคิด "Divide and Conquer" หรือการแบ่งแยกและเอาชนะ เพื่อจัดการข้อมูลขนาดใหญ่ได้อย่างมีประสิทธิภาพโดยการย่อยปัญหาให้เล็กลงกลไกการทำงาน ทำการหั่นอาเรย์ออกเป็นสองซีกซ้ำๆ จนเหลือสมาชิกตัวเดียว จากนั้นจึงทำการ Merge หรือรูดซิปกลับเข้าด้วยกันอย่างเป็นระเบียบจุดเด่น เป็นอัลกอริทึมที่เสถียร (Stable) ข้อมูลที่เท่ากันจะรักษาลำดับเดิมไว้ และรับประกันความเร็วที่ O(n \log n) เสมอไม่ว่าข้อมูลจะแย่แค่ไหนข้อเสีย จำเป็นต้องใช้หน่วยความจำสำรอง (Extra Space) ในการพักข้อมูลระหว่างกระบวนการรวมไฟล์//

**Quick Sort นักซิ่งทำลายสถิติ**//อัลกอริทึมที่เน้นความเร็วสูงสุดในสถานการณ์ส่วนใหญ่ โดยใช้กลไกการเลือกจุดหมุนเพื่อจัดกลุ่มข้อมูลกลไกการทำงาน ใช้การเลือก Pivot (จุดหมุน) แล้วแยกข้อมูลที่น้อยกว่าไว้ฝั่งซ้ายและมากกว่าไว้ฝั่งขวา จากนั้นทำซ้ำกระบวนการเดิมกับข้อมูลทั้งสองฝั่งจุดเด่น ทำงานแบบ In-place ไม่ต้องใช้หน่วยความจำเพิ่มมหาศาล และมีความเป็น Cache Friendly สูง ทำให้ในทางปฏิบัติมักเร็วกว่า Merge Sortข้อเสีย หากเลือก Pivot ไม่เหมาะสมในข้อมูลที่เรียงลำดับมาอยู่แล้ว ความเร็วอาจลดลงไปถึง O(n^2)//

**ตารางเปรียบเทียบความแรง (Big O Complexity)**//การเปรียบเทียบประสิทธิภาพเชิงเวลาและหน่วยความจำของอัลกอริทึมแต่ละประเภทเพื่อให้เลือกใช้งานได้ถูกต้องAlgorithmAverage CaseWorst CaseSpace Complexityจุดเด่นBubble SortO(n^2)O(n^2)O(1)เขียนง่าย แต่ช้ามากMerge SortO(n log n)O(n log n)O(n)เสถียรและแน่นอนQuick SortO(n log n)O(n^2)O(log n)เร็วที่สุดในงานส่วนใหญ่//

**การเลือกใช้งานในโลกนักพัฒนา**//การทำความเข้าใจกลไกภายในช่วยให้เลือกใช้ฟังก์ชันมาตรฐานในภาษาโปรแกรมต่างๆ ได้อย่างเหมาะสมตามลักษณะของงานJava และ Python มักใช้ Timsort ซึ่งเป็นลูกผสมระหว่าง Merge และ Insertion Sort เพื่อเน้นความเสถียรกับข้อมูลในโลกความจริงC++ ในฟังก์ชัน std::sort มักใช้ Introsort ที่เริ่มต้นด้วย Quick Sort แต่จะสลับไปใช้ Heap Sort หากประสิทธิภาพเริ่มลดลงเพื่อป้องกันกรณี Worst Case//

บทที่ 9: External Storage (การจัดเก็บข้อมูลภายนอกและโครงหลังคาฐานข้อมูล)
**B-Tree และ B+ Tree โครงสร้างข้อมูลเพื่อระบบจัดเก็บข้อมูลขนาดใหญ่**
//ในโลกของการประมวลผลที่มีข้อมูลระดับ Terabyte ซึ่งไม่สามารถเก็บใน RAM ได้ทั้งหมด การอ่านข้อมูลจาก Secondary Storage (SSD/Hard Disk) จึงเป็นคอขวดสำคัญเนื่องจากมีความช้ากว่า RAM มหาศาล โครงสร้างแบบ B-Tree จึงถูกออกแบบมาเพื่อลดจำนวนการเข้าถึงดิสก์ให้เหลือน้อยที่สุด
ปัญหาของต้นไม้ทั่วไปในโลกของ Disk การใช้ Binary Search Tree ปกติจะทำให้ต้นไม้สูงเกินไป ส่งผลให้เกิด Disk Latency จากการต้องอ่านดิสก์หลายครั้ง และเสียพื้นที่ใน Disk Block ไปโดยเปล่าประโยชน์เนื่องจากหนึ่งโหนดเก็บข้อมูลได้น้อยเกินไป//

**B-Tree ยิ่งอ้วน ยิ่งดี**
//ทางออกคือการสร้างต้นไม้ที่มีลักษณะเตี้ยและกว้าง เพื่อให้ความสูงของต้นไม้ลดลงและเก็บข้อมูลได้พอดีกับขนาดหนึ่ง Disk Block
Multi-way Tree ยอมให้หนึ่งโหนดเก็บคีย์ได้หลายสิบค่าและมีลูกได้หลายร้อยตัว ทำให้ความสูงของต้นไม้ลดลงมหาศาล
Balanced ระบบจะรักษาความสมดุลเสมอโดยการเติมข้อมูลให้เต็มโหนดก่อนทำการแตกตัว (Split)
Efficiency ช่วยให้การค้นหาข้อมูลมหาศาลใช้การอ่านดิสก์ (Disk Accesses) เพียงไม่กี่ครั้งเท่านั้น//

**B+ Tree มาตรฐานระดับอุตสาหกรรม**
//ร่างสมบูรณ์ของ B-Tree ที่ถูกนำไปใช้ในระบบฐานข้อมูลสมัยใหม่ เช่น MySQL และ PostgreSQL โดยปรับปรุงประสิทธิภาพการค้นหาให้ดียิ่งขึ้น
Leaf Nodes Only ข้อมูลจริงจะถูกเก็บไว้ที่โหนดใบเท่านั้น ส่วนโหนดด้านบนจะเก็บเพียงคีย์ดัชนีเพื่อใช้ในการบอกทาง
Linked Leaves โหนดใบทั้งหมดจะเชื่อมโยงกันด้วย Linked List ทำให้การค้นหาข้อมูลแบบเป็นช่วง (Range Query) ทำได้รวดเร็วเพราะสามารถเดินตามรายการไปได้ทันทีโดยไม่ต้องย้อนกลับไปที่รากใหม่
การประยุกต์ใช้ เป็นโครงสร้างหลักเบื้องหลังระบบ File System และ Database Indexing ที่เราใช้งานกันในปัจจุบัน//

บทที่ 10: การคัดเลือกโครงสร้างข้อมูล (The Art of Decision Making)
**วิเคราะห์ด้วย Big O เข็มทิศของนักพัฒนา**//ก่อนจะเลือกโครงสร้างข้อมูล เราต้องประเมินพฤติกรรมของข้อมูลในระบบก่อนโดยใช้ Big O Notation เป็นตัววัด เพื่อให้เข้าใจถึงประสิทธิภาพเชิงเวลาและพื้นที่Time Complexity พิจารณาว่าระบบเน้นการค้นหา (Search), การเพิ่มหรือลบ (Insert/Delete), หรือการเรียงลำดับ (Sort) เป็นหลักSpace Complexity ประเมินข้อจำกัดของหน่วยความจำ เช่น ในระบบ Embedded ที่มี RAM จำกัด หรือระบบ Server ที่เน้นความเร็วสูงสุดโดยยอมเสียสละพื้นที่//

**Decision Matrix ตารางคัดเลือกอาวุธ**//สรุปการตัดสินใจเลือกโครงสร้างข้อมูลตามประสิทธิภาพของปฏิบัติการหลัก เพื่อให้เลือกใช้งานได้ตรงตามความต้องการของระบบโครงสร้างข้อมูลSearchInsert/Deleteจุดเด่นเหมาะสำหรับArrayO(1)O(n)เข้าถึงตำแหน่งแน่นอนได้เร็วที่สุดข้อมูลที่ขนาดคงที่และเน้นการอ่านLinked ListO(n)O(1)ยืดหยุ่น เพิ่มลดตรงกลางได้ง่ายงานคิวหรือรายการที่เปลี่ยนขนาดบ่อยHash TableO(1)O(1)เร็วที่สุดถ้าไม่มีการชนระบบค้นหาชื่อ, Cache, Database IndexBalanced BSTO(log n)O(log n)รักษาลำดับข้อมูลและค้นหาได้เร็วระบบจัดการไฟล์, ข้อมูลที่ต้องเรียงตลอดเวลาBinary HeapO(n)O(\log n)หาค่าสูงสุด/ต่ำสุดได้ใน O(1)Task Scheduling, Priority Queue//

**การออกแบบเชิงสถาปัตยกรรม (Composite Structures)**//วิศวกรซอฟต์แวร์มักนำโครงสร้างข้อมูลหลายแบบมาผสมผสานกันเพื่อให้ได้ประสิทธิภาพสูงสุดในทุกมิติLRU Cache (Least Recently Used) การผสมผสานระหว่าง Hash Table เพื่อการค้นหาที่รวดเร็ว และ Doubly Linked List เพื่อจัดการลำดับข้อมูลเก่า-ใหม่Social Network Graph การใช้ Adjacency List เก็บเครือข่ายความสัมพันธ์ โดยภายในอาจใช้ Hash Set เพื่อตรวจสอบสถานะความเป็นเพื่อนได้ทันที//

**Trade-offs สิ่งที่ต้องแลกเพื่อให้ได้มา**//การปรับแต่งประสิทธิภาพมักมาพร้อมกับทางเลือกที่ต้องเสียสละอย่างใดอย่างหนึ่งเสมอTime vs. Space การยอมใช้หน่วยความจำเพิ่มขึ้น เช่น การทำ Caching หรือ Indexing เพื่อลดภาระการประมวลผลของ CPU ให้ทำงานได้เร็วขึ้นComplexity vs. Maintenance อัลกอริทึมที่ซับซ้อนอย่าง Red-Black Tree อาจให้ความเร็วสูงสุดแต่ก็แลกมาด้วยความยากในการดูแลรักษาและเสี่ยงต่อการเกิดข้อผิดพลาดมากกว่าโครงสร้างพื้นฐาน//
